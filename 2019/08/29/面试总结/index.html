面试总结

Posted on 2019-08-29 | In 日志

前言

对我每一次的面试经历进行分析，总结出自己的不足和面试的重点。
以前觉得这样做很浪费时间，但是真的在总结的时候就发现原来自己真的好多问题，希望自己可以坚持下去！

2019.08.25 北京纬创(字节跳动，驻场)

问题：

  1. vue用了多久？最近半年的工作内容？

    用了两年多；利用微服务的概念，将以前的项目进行重构，具体是将原有项目(品控管理)按照模块及功能进行拆分，拆分出来的每一块就作为微服务的模块之一。这样做的好处是：1.不会影响其他模块、2.可以快速迭代开发、3.项目比较大时，整个项目的打包及部署都很慢，影响后续的开发工作，而微服务是轻便快捷的；

  2. 微服务相关的项目？在项目中负责的模块及团队规模？

    （当前简历中未注明该项目）
    大后台管理；主要负责其中的用户管理及角色管理模块，后期也帮其他同事做过其他模块(工队管理、项目组成员等)；

  3. 有关vue及后台业务管理的开发经验，在项目中做的比较好的可以讲一下(主要想要快速了解我的技术能力)。

    （我的理解是，遇到了什么问题是怎么解决的；回答：开发时间比较紧、只做了项目迁移、产品原型不清晰，开发之前未考虑清楚，跟产品需要沟通）
    目前来讲主要还是使用vue，所以开发中elementUI、vue-Router、vuex用的还是比较多比较熟练的。遇到问题比较多的地方是组件的使用，或者说是在原项目中有些泛滥成灾的组件，特别是组件与组件的层层嵌套，有时候会很难去找到组件的文件。这种使用方法，违背了组件使用的初衷，所以我现在在开发时，会考虑是否需要将一个功能以组件的形式编写，还有一种情况是页面模块按照组件拆分，这个是可以是页面的代码更加清晰明确不会因为复杂的功能而使页面的代码变得冗长难读。
    ——上面表述了我在原项目中看到的问题，我的思考，但是并未解决。

  4. 封装组件原因、内容及他人如何使用？

    组织架构的弹窗：这个弹窗是用来让用户选择其所在公司组织架构下的部门的，因为在当前后台管理项目中多个页面中都有使用且样式和功能都是一致的仅数据不同，所以将其封装为一个组件，方便别人的使用。因为这个弹窗被封装在一个单独的vue文件内，所以只要将这个文件引入相应的页面(import ‘file’ from ‘components.vue’)中，并且注册后即可使用。
    还有些弹窗需要根据组织架构来选择其某个部门下的人员，所以在组织架构弹窗的基础上进行扩展，在选择了当前部门最后一级时，可以再去加载其部门下对应的人员。
    这部分的难点可能是数据的回显，已选择的数据的复选框应该是勾选状态的(不可再次勾选)，并且在列表右侧一列展示出当前已选中的数据，并且可以取消选中。之所以说是难点是因为使用v-modal来绑定已选中数据的，所以接口返回的选中数据，需要与原数据的结构保持一致，这一部分需要与后端沟通，告诉他所需的字段且不可随意更改。如果接口是已经写好的，并且有多余字段的存在，可以在获取数据后进行处理，获取自己所需要的字段即可。

  5. vue中的set函数(this.$set)在什么场景下使用？使用的具体场景是什么(在项目中，哪个功能用到这个方法)？

    (接口返回的数据，对数据进行处理；正常情况下，按照下标为每一个元素赋值时，不会生效就需要用这个方法；做微服务时有个新模块叫角色管理，获取角色列表后需要勾选中被选择的数据，因为是使用v-model来实现选中。这里面有一个字段来判断是否选中，但是返回的数据是没有的，所以自己重新处理的时候用这个方法)
    对数组进行处理

    Vue.set( target, propertyName/index, value )
    参数：
    {Object | Array} target
    {string | number} propertyName/index
    {any} value
    返回值：设置的值。
    用法：
    向响应式对象中添加一个属性，并确保这个新属性同样是响应式的，且触发视图更新。它必须用于向响应式对象上添加新属性，因为 Vue 无法探测普通的新增属性 (比如 this.myObject.newProperty = 'hi')

    (面试官：技术方面主要在使用层，原理性的知识知道的比较少，往深了问问不下去。有感觉到在看一些相关的知识，由于原有项目的原因，没有接触到比较复杂的场景，所以总体来说，是面不过的。如果公司里不具备去学习或者使用新技术的环境，可以主动带头去做一些变更，在原有公司会有提升。主要还是需要学习的)

自我总结：把面试中的录音听了一遍，发现了很多问题。

  • 在回答问题时，理解问题有困难，总是需要想一下具体问的是什么问题。
  • 有口头语，额、啊、 这个听起来特别不好。可以思考然后准确说出想说的答案，不带有‘好像’、‘应该’这些词。
  • 说话逻辑不清晰，说着说着就乱了，这点特别不好。
  • 非常没有自信心
  • 主要还是原理性的问题比较多，还是得多看面试题，多看一些原理性的东西。
  • 别着急，别对自己失望，慢慢来，好好复习，这次一定要找一家好一点的公司。
